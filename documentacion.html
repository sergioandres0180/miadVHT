Detalles del proceso ETL, arquitectura y fuentes de datos del proyecto.

1. Resumen del Proyecto

Este proyecto consiste en un dashboard interactivo construido en Looker Studio para analizar y visualizar datos salariales de profesionales a nivel global, obtenidos a través de la encuesta anual de Ask a Manager (2021). El objetivo es facilitar la comparación de salarios por industria, país, género, nivel de educación y años de experiencia.

La arquitectura del proyecto se compone de tres etapas principales:

  • Extracción de datos crudos desde la encuesta pública de Ask a Manager (Google Sheets).
  • Limpieza y transformación de los datos mediante un Jupyter Notebook en Python.
  • Carga de los datos limpios a Google Sheets y conexión con Looker Studio para la visualización.

Sitio web en producción: https://grupo16-eight.vercel.app/

2. Arquitectura del Sistema

El flujo de datos sigue una pipeline ETL (Extract, Transform, Load) simplificada:

Etapa Herramienta Recurso Descripción
1. Extract Google Sheets Ask a Manager Survey 2021 Respuestas crudas de la encuesta (~28,000 filas)
2. Transform Python / Jupyter limpieza_sueldos.ipynb Normalización de países, monedas, cargos. Conversión a USD/COP.
3. Load Excel → Google Sheets sueldos_limpio2.xlsx Datos limpios cargados como fuente para Looker Studio
4. Visualize Looker Studio Dashboard embebido Gráficos interactivos y filtros dinámicos

4. Proceso de Limpieza de Datos

El notebook limpieza_sueldos.ipynb ejecuta la pipeline de transformación en los siguientes pasos:

4.1 Librerías Requeridas

Instalar antes de ejecutar el notebook:

pip install pandas numpy openpyxl thefuzz pycountry forex-python

4.2 Pasos del Pipeline

Paso 1 — Carga de datos: Se carga el archivo Excel crudo descargado del Google Sheets de Ask a Manager:

df = pd.read_excel('SUELDOS PROFESIONALES.xlsx')

Paso 2 — Normalización de países (country_clean): La columna de país contiene texto libre con errores tipográficos, siglas, nombres locales y entradas irrelevantes. Se aplica una función de tres pasos:

  • Diccionario de mapeo manual (siglas_map): cubre siglas (US, UK, NZ, etc.) y nombres alternativos.
  • Coincidencia exacta contra la lista de países de la librería pycountry.
  • Fuzzy matching con umbral de 85% usando thefuzz para capturar errores ortográficos menores.

Los valores que no superan el umbral se convierten en NaN y son excluidos del análisis.

Paso 3 — Normalización de moneda (currency_clean): Se estandariza a mayúsculas y se eliminan registros con moneda 'OTHER'.

df['currency_clean'] = df['Please indicate the currency'].str.strip().str.upper()

Paso 4 — Conversión de salarios a COP: Se obtienen tasas de cambio en tiempo real usando forex_python para las principales monedas. Se crean tres columnas nuevas: salario_anual_COP, compensaciones_COP y total_COP.

Paso 5 — Creando nuevas columnas: Se crearon nuevas columnas para no borrar la información original:

Nombre Información
Pais_Limpio Resultado de la limpieza de la columna de países.
Ciudad_Limpia Resultado de la limpieza de la columna de ciudades.
Industria_Clasificada Agrupación general de sectores económicos para reducir la granularidad.
ID Identificador único de registro.

Paso 6 — Normalización de la columna etnia: Se separan los datos por comas, generando nuevas filas para registros con múltiples etnias. Esto permite que en Looker Studio las agregaciones a nivel de "ID" funcionen correctamente mientras se visualizan todos los detalles de etnia.

Paso 7 — Traducción: Proceso de traducción al español de las columnas de etnia, sexo, nivel de escolaridad e industria clasificada para un mejor entendimiento del reporte por usuarios colombianos.

Paso 8 — Exportación: El dataframe limpio se exporta en dos formatos:

df_clean.to_excel('sueldos_limpio2.xlsx', index=False)
df_clean.to_csv('sueldos_limpio.csv', index=False, encoding='utf-8-sig')

5. ¿Cómo Agregar o Actualizar Datos?

Para actualizar el dashboard con nuevos datos (por ejemplo, una edición posterior de la encuesta Ask a Manager), seguir este procedimiento:

5.1 Actualización desde la misma fuente

  • Descargar el Google Sheet actualizado de Ask a Manager como archivo Excel (.xlsx).
  • Guardar el archivo con el nombre SUELDOS PROFESIONALES.xlsx en la ruta que lee el notebook (Cell 1).
  • Ejecutar todas las celdas del notebook limpieza_sueldos.ipynb de arriba a abajo.
  • Verificar que no hay errores en la ejecución, especialmente en la conversión de monedas (requiere conexión a internet para forex_python).
  • El notebook generará un nuevo archivo sueldos_limpio2.xlsx listo para subir.

5.2 Carga a Google Sheets

  • Abrir el Google Sheet que sirve como fuente en Looker Studio.
  • Eliminar todos los datos existentes (mantener la fila de encabezados).
  • Copiar y pegar el contenido del archivo sueldos_limpio2.xlsx en la hoja, o usar Archivo → Importar para reemplazar los datos.
  • Looker Studio se actualiza automáticamente en la próxima carga del dashboard.

5.3 Agregar nuevas columnas

Si se desea agregar nuevas variables al análisis:

  • Agregar la transformación correspondiente en el notebook antes de la celda de renombrado (Cell 11).
  • Incluir el nuevo nombre de columna en el diccionario de rename().
  • Volver a exportar el archivo Excel y reemplazar la fuente en Google Sheets.
  • En Looker Studio, ir a Recurso → Gestionar fuentes de datos y actualizar los campos disponibles.

6. ¿Cómo Modelar Nuevas Entradas?

Si se desea incorporar datos de una fuente distinta (por ejemplo, otra encuesta de salarios o datos propios), seguir estas indicaciones para garantizar compatibilidad con el pipeline existente:

6.1 Estructura mínima requerida

El nuevo dataset debe contener las siguientes columnas (puede usar los nombres originales o los nombres limpios):

Campo requerido Tipo Notas
country (o country_clean) Texto Si está sucio, pasarlo por la función limpiar_pais().
annual_salary Numérico En moneda local. Anualizado si es por hora.
currency_clean Texto (código ISO) USD, EUR, GBP, CAD, etc. En mayúsculas.
Industry Texto Categoría de industria.
job_title Texto Normalizar con .str.title().
experience_total / experience_field Texto categórico Rangos: '1-2 years', '5-7 years', etc.
education, gender, race Texto categórico Opcionales para análisis demográfico.

6.2 Reutilización del pipeline

Las funciones del notebook son modulares y reutilizables:

  • limpiar_pais(valor): recibe un string y retorna el nombre estandarizado del país (o NaN si no reconoce).
  • convertir_a_usd(row, col_salario): convierte el salario de la fila a USD usando las tasas precargadas en tasas_a_usd.

Para adaptar el notebook a una nueva fuente:

  • Cambiar la ruta del archivo en Cell 1.
  • Ajustar los nombres de columna en los parámetros de las funciones apply().
  • Actualizar el diccionario rename() con los nuevos nombres de columna si cambian.

6.3 Agregar nuevas monedas

Si la nueva fuente incluye monedas no contempladas actualmente, agregar la tasa al diccionario en Cell 7:

tasas_a_usd['COP'] = c.get_rate('COP', 'USD') # Ejemplo: Peso colombiano

9. Consideraciones Técnicas y Advertencias

  • Dependencia externa: Tasas de cambio dinámicas
    La función get_rate() de forex_python requiere conexión a internet. Si falla, precargar las tasas manualmente en el diccionario tasas_a_usd como valores fijos.
  • Compatibilidad futura: Cambios en la encuesta
    Si Ask a Manager modifica los nombres de las columnas en ediciones futuras, actualizar el diccionario rename() en la Cell 11 del notebook.
  • Ajuste de parámetros: Umbral de fuzzy matching
    El umbral de 85% en limpiar_pais() puede ajustarse. Un umbral menor incluye más países pero puede generar falsos positivos. Un umbral mayor es más estricto.
  • Criterio estadístico: Umbral de agrupación racial
    El umbral de 60 ocurrencias para agrupar razas en 'Other' puede modificarse según el tamaño del nuevo dataset.
  • Filtrado aplicado: Registros excluidos
    Se excluyen del análisis todos los registros con moneda 'OTHER' y aquellos donde el país no pudo ser identificado. Revisar el output de la Cell 6 para conocer el impacto de estos filtros.
